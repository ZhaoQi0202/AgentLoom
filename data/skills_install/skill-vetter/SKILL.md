# skill-vetter

## 何时使用

- 用户或流程在**安装、启用、合并**某技能前需要审计。
- `find-skills` 返回候选后，进入本技能做**门禁**。

## 安装目标核对

- **任务级**（find-skills）：`<任务根>/.agentloom/skills/<id>/`。  
- **应用级**（技能管理 / 内置默认）：`data/skills_install/<id>/`。

## 审查清单（逐项在输出中体现结论）

1. **结构**  
   - 存在可读 `SKILL.md`；建议有 YAML frontmatter（至少 `name`、`description`）。

2. **安全**  
   - 是否要求无提示执行 Shell、访问网络、读写用户目录外路径、窃取密钥。  
   - 是否包含混淆、加密载荷或不可审计的二进制指令。

3. **质量**  
   - 步骤是否可执行、术语是否与当前任务上下文一致；是否存在未声明的外部依赖。

4. **许可与来源**  
   - 若可识别，注明许可证或来源；未知则标明「来源未验证」。

## 输出格式

- **结论**：`通过` | `有条件通过` | `不通过`  
- **摘要**：3～6 条要点。  
- **风险项**：分级（高/中/低）与缓解建议。  
- **若「不通过」**：明确阻止安装/启用，并说明需用户或架构师决策的例外条件。

## 禁止

- 为通过审查而隐瞒或弱化已识别的高风险行为。
- 代替用户做最终采纳决定（最终决定仍为人或上层流程）。
